SECTION 1: ASSEMBLY INSTRUCTIONS
═══════════════════════════════════════════════════════════════════════════════

STEP 1: PREPARE HARDWARE
───────────────────────────────────────────────────────────────────────────────

Required Components:
  [ ] Arduino Mega2560 (or compatible)
  [ ] MCP_CAN Shield (MCP2515-based CAN controller)
  [ ] CAN Bus Cables (CAN_H, CAN_L, GND)
  [ ] 120Ω Termination Resistors (2 total)
  [ ] Power Supply (7-12V DC, 1-2A capacity)
  [ ] Sensors and actuators for your application
  [ ] USB Cable (for firmware upload)

STEP 2: INSTALL MCP_CAN SHIELD
───────────────────────────────────────────────────────────────────────────────

BEFORE INSTALLATION:
┌──────────────┐
│   Arduino    │
│   Mega2560   │
│              │
│  [Pins]      │
│  [Pins]      │
│  [Pins]      │
└──────────────┘

INSTRUCTIONS:
  1. Power OFF Arduino (disconnect USB and power supply)
  2. Align shield pins with Arduino header pins
  3. Check for bent pins - straighten if needed
  4. Press shield down firmly until it sits flush
  5. Verify shield is properly seated (no gaps)

AFTER INSTALLATION:
┌──────────────┐
│   Arduino    │
│   Mega2560   │
│              │
│  ┌─────────┐│
│  │  Shield ││
│  │  MCP2515││
│  │  CAN    ││
│  └─────────┘│
│              │
└──────────────┘

VERIFICATION:
  [ ] Shield sits flush on Arduino (no gaps)
  [ ] All pins are properly connected
  [ ] No bent or damaged pins
  [ ] Shield power LED lights up (if present)

STEP 3: CONNECT CAN BUS
───────────────────────────────────────────────────────────────────────────────

CAN BUS TOPOLOGY:
                                CAN BUS
                                   │
        ┌──────────────────────────┼──────────────────────────┐
        │                          │                          │
        ▼                          ▼                          ▼
┌──────────────┐          ┌──────────────┐          ┌──────────────┐
│ epicEFI ECU  │          │  Arduino     │          │ Other CAN     │
│              │          │  Mega2560    │          │ Devices       │
│  CAN_H ──────┼──────────┼──► CAN_H     │          │ CAN_H ────────┤
│  CAN_L ──────┼──────────┼──► CAN_L     │          │ CAN_L ────────┤
│  GND   ──────┼──────────┼──► GND       │          │ GND   ────────┤
│              │          │              │          │               │
│ [120Ω]       │          │              │          │ [120Ω]        │
│ Terminator   │          │              │          │ Terminator    │
└──────────────┘          └──────────────┘          └──────────────┘
    (Node 1)                  (Node 2)                  (Node 3)

WIRING STEPS:
  1. Connect CAN_H:
     Shield CAN_H ──────────────────► ECU CAN_H
     
  2. Connect CAN_L:
     Shield CAN_L ──────────────────► ECU CAN_L
     
  3. Connect Ground:
     Shield GND   ──────────────────► ECU GND
     (Common ground is REQUIRED)
     
  4. Install Termination Resistors:
     [120Ω] at Shield end (if not already on shield)
     [120Ω] at ECU end
     
     IMPORTANT: Only TWO termination resistors total!

VERIFICATION:
  [ ] CAN_H connected (twisted pair with CAN_L recommended)
  [ ] CAN_L connected
  [ ] GND connected (common ground)
  [ ] Exactly 2 termination resistors installed
  [ ] No extra termination resistors

  [ ] Power LED on Arduino lights up
  [ ] Shield power LED lights up (if present)

LOGIC:
  Button NOT pressed: D20-D34 = HIGH (pullup active) = 0 (inactive)
  Button pressed:      D20-D34 = LOW (grounded)      = 1 (active)

BITFIELD MAPPING:
  Bit 0  = D20
  Bit 1  = D21
  Bit 2  = D22
  ...
  Bit 14 = D34

WIRING CHECKLIST:
  [ ] Button Terminal 1 connected to D20-D34
  [ ] Button Terminal 2 connected to GND
  [ ] Verify button works (test with multimeter)

  Connect across relay coil (cathode to +12V side)
  Prevents voltage spike when relay turns off

WIRING CHECKLIST:

  [ ] Flyback diode across relay coil
  [ ] Transistor emitter to GND
  [ ] Relay coil terminals: one to transistor collector, one to +12V
  [ ] Load connected to relay contacts (not shown, depends on application)

PWM CHARACTERISTICS:
  Frequency: ~490Hz (D2-D13) or ~980Hz (D44-D46)
  Resolution: 8-bit (0-255 = 0-100%)
  Duty Cycle: 0% = motor stopped, 100% = motor full speed

PWM PIN ASSIGNMENTS:
  D2, D3, D4, D5, D6, D7, D8      (7 pins, ~490Hz)
  D10, D11, D12, D13               (4 pins, ~490Hz)
  D44, D45, D46                    (3 pins, ~980Hz)
  Total: 14 PWM outputs

WIRING CHECKLIST:
  [ ] PWM signal connected to motor driver input
  [ ] Motor driver power supply connected (12V typical)
  [ ] Motor connected to driver outputs
  [ ] Ground connections verified

  Maximum Length: ~40 meters at 500 kbps
  Cable Type: Twisted pair recommended
  Impedance: ~60Ω with termination (120Ω at each end)

TERMINATION:
  Exactly TWO 120Ω resistors total
  One at Arduino/shield end
  One at ECU end
  Creates 60Ω total bus impedance

SECTION 6: TESTING & VERIFICATION
═══════════════════════════════════════════════════════════════════════════════

PRE-POWER CHECKLIST
───────────────────────────────────────────────────────────────────────────────

Before applying power, verify:
  [ ] Shield properly installed on Arduino
  [ ] No bent or damaged pins
  [ ] CAN bus wiring correct (CAN_H, CAN_L, GND)
  [ ] Termination resistors installed (exactly 2, 120Ω each)
  [ ] Power supply voltage correct (7-12V DC)
  [ ] Power supply polarity correct (positive to VIN)
  [ ] All I/O connections verified (if connected)
  [ ] No short circuits
  [ ] Ground connections verified

POWER-ON VERIFICATION
───────────────────────────────────────────────────────────────────────────────

1. Connect power supply
   [ ] Arduino power LED lights up
   [ ] Shield power LED lights up (if present)

2. Connect USB cable for Serial Monitor
   [ ] USB connection established
   [ ] Serial Monitor opens at 115200 baud

3. Check Serial Output:
   Expected messages:
     "CAN BUS OK!"
     "Pin initialization complete"
     "=== Feature Status ==="

4. Verify CAN Bus:
   [ ] No "CAN BUS FAIL!" errors
   [ ] Serial shows "CAN BUS OK!"

5. Verify ECU Communication:
   [ ] ECU is powered and running
   [ ] Wait for ECU response (may take a few seconds)
   [ ] Should see "ECU communication restored" when ECU responds

FUNCTIONAL TESTING
───────────────────────────────────────────────────────────────────────────────

1. Enable Serial Commands:
   Edit firmware: #define ENABLE_SERIAL_COMMANDS true
   Upload firmware

2. Test Analog Inputs:
   Serial Monitor → Send: TESTANALOG
   [ ] All 16 channels read values (check voltage with multimeter)

3. Test Digital Inputs:
   Serial Monitor → Send: TESTDIGITAL
   [ ] All inputs show state (press buttons, verify state changes)

4. Test Digital Outputs:
   Serial Monitor → Send: TESTOUTPUTS
   [ ] Outputs cycle through test pattern
   [ ] Visual verification (LEDs, relays clicking, etc.)

5. Test PWM Outputs:
   Serial Monitor → Send: TESTPWM
   [ ] Each PWM channel sweeps from 0% to 100%
   [ ] Visual/scope verification (motor speed, LED brightness, etc.)

6. Check Performance:
   Serial Monitor → Send: STATS
   [ ] CAN TX rate > 0
   [ ] CAN RX rate > 0 (if ECU connected)
   [ ] Error rate < 1%

TROUBLESHOOTING QUICK REFERENCE
───────────────────────────────────────────────────────────────────────────────

Problem: "CAN BUS FAIL!"
  → Check SPI connections (MOSI, MISO, SCK, CS)
  → Verify MCP2515 power supply
  → Check shield seating

Problem: "No ECU response"
  → Verify ECU_CAN_ID matches ECU
  → Check CAN bus wiring (CAN_H, CAN_L, GND)
  → Verify termination resistors (2 total)
  → Ensure ECU is powered and transmitting

Problem: Analog inputs wrong values
  → Verify sensor voltage range (0-5V)
  → Check for floating inputs
  → Verify ground connections
  → Enable ADC calibration if needed

Problem: Outputs not responding
  → Check variable hashes configured
  → Verify ECU communication
  → Check Serial output for errors
  → Test outputs manually with serial commands

See TROUBLESHOOTING.md for detailed solutions.
